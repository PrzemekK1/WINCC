◆ FW_INSTALLATION_REDU_INFO_DP

const string FW_INSTALLATION_REDU_INFO_DP = "fwReduInfo"

Name of user-supplied DP that parametrizes the result of checking if system is redundant. If user creates fwReduInfo DP in a system and sets the isRedundant DPE, then fwInstallationRedu functions that checks whether project is redundant will return result as given in isRedundant DPE without performing any additional validations. This affects: fwInstallationRedu_isRedundant() and fwInstallationRedu_isSystemRedundant()

Purpose: Verification whether project is redundant or not may be impossible in certain states and and configurations - for example in remote system when redu manager is stopped there or when running a scattered manager where redu connection parameters are supplied in the command argument and not in the config. User can use fwReduInfo DP to pernamently mark system as redundant so that fwInstallationRedu functions will always return the same result, regardless of project state and manager configuration.

IMPORTANT: Once fwReduInfo DP is created by user, it becomes its responsibility to ensure correctness of information supplied in that DP!

To create fwReduInfo DP: 1) Create DPT FwReduInfo containing one boolean DPE: isRedundant FwReduInfo [DPEL_STRUCT] |— isRedundant [DPEL_BOOL] 2) Create datapoint fwReduInfo 3) Set isRedundant DPE to desired value (TRUE if redundant system, FALSE if non-redu)
